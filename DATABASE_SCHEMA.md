# Database Schema for QA Generator

## Current Setup: Firestore (Google Firebase)

### Collection: `products`

```
Collection: products
├── Document ID: <auto-generated by Firebase>
└── Fields:
    ├── id (string/number) - Document ID
    ├── uniqueId (string) - Unique product identifier
    ├── name (string) - Product name
    ├── batch (string) - Batch number
    ├── mfg (string) - Manufacturing date
    ├── expiry (string) - Expiry date
    ├── shortUrl (string) - QR code short URL
    ├── manufacturer (string) - Manufacturer name (optional)
    ├── manufacturerAddress (string) - Manufacturer address (optional)
    ├── technicalName (string) - Technical/chemical name (optional)
    ├── registrationNumber (string) - Registration number (optional)
    ├── packingSize (string) - Packing size (optional)
    ├── manufacturerLicence (string) - License number (optional)
    └── createdAt (timestamp) - Auto-created timestamp
```

### Option 2: Amazon RDS (PostgreSQL/MySQL)

**Table Name:** `products`

```sql
CREATE TABLE products (
  id SERIAL PRIMARY KEY,
  unique_id VARCHAR(50) NOT NULL UNIQUE,
  name VARCHAR(255) NOT NULL,
  batch VARCHAR(100) NOT NULL,
  mfg VARCHAR(50) NOT NULL,
  expiry VARCHAR(50) NOT NULL,
  short_url VARCHAR(500),
  manufacturer VARCHAR(255),
  manufacturer_address TEXT,
  technical_name VARCHAR(255),
  registration_number VARCHAR(100),
  packing_size VARCHAR(100),
  manufacturer_licence VARCHAR(100),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  deleted_at TIMESTAMP NULL,
  
  INDEX idx_unique_id (unique_id),
  INDEX idx_manufacturer (manufacturer),
  INDEX idx_created_at (created_at)
);
```

## Field Descriptions

| Field | Type | Required | Length | Description |
|-------|------|----------|--------|-------------|
| id | String | Optional | - | Unique document/record ID |
| uniqueId | String | Yes | 50 | Unique product identifier (timestamp-based) |
| name | String | Yes | 255 | Product name/title |
| batch | String | Yes | 100 | Batch number for tracking |
| mfg | String | Yes | 50 | Manufacturing date (MM/YYYY format) |
| expiry | String | Yes | 50 | Expiry date (MM/YYYY format) |
| shortUrl | String | Optional | 500 | QR code URL/short URL |
| manufacturer | String | Optional | 255 | Manufacturer company name |
| manufacturerAddress | String | Optional | - | Full address of manufacturer |
| technicalName | String | Optional | 255 | Chemical/technical name |
| registrationNumber | String | Optional | 100 | Regulatory registration number |
| packingSize | String | Optional | 100 | Package size (e.g., 500ml) |
| manufacturerLicence | String | Optional | 100 | License number |
| createdAt | Timestamp | Auto | - | Record creation timestamp |
| updatedAt | Timestamp | Auto | - | Last update timestamp |
| deletedAt | Timestamp | Optional | - | Soft delete timestamp (for recovery) |

## Migration Path: Firestore → Amazon DB

### Step 1: Export from Firestore
```javascript
// Export collection as JSON
const snapshot = await getDocs(collection(db, 'products'));
const data = [];
snapshot.forEach(doc => {
  data.push({ id: doc.id, ...doc.data() });
});
// Save to JSON file
```

### Step 2: Transform Data
```javascript
// Transform snake_case for RDS
const transformed = data.map(product => ({
  unique_id: product.uniqueId,
  name: product.name,
  // ... map all fields
}));
```

### Step 3: Import to Amazon DB
- Use AWS Data Migration Service (DMS)
- Or bulk insert via SQL/API

### Step 4: Update database.ts
Replace Firebase methods with Amazon DB API calls
